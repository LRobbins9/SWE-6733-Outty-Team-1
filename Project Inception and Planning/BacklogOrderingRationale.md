# Backlog Ordering Rationale  
The backlog contains 11 work items that are ordered from highest priority to lowest priority. An ordering of the items can be seen in the figure below.  

<img width="1437" height="622" alt="image" src="https://github.com/user-attachments/assets/37fc4449-3210-4ac6-8ad5-92c708be3b16" />




The backlog items start with the highest priority work items. A breakdown of the priority and rationale can be seen in the table below.  

### Backlog Odering Table  

| Work Item | Description | Priority | Rationale |
| :--- | :--- | :--- | :--- |
| SCRUM-11 | Choose Technology stack | Highest | The technology stack must be the first thing that’s chosen since that’s what the app will be built on. |
| SCRUM-31 | Setup User Database | Highest | The user database is of high importance since it will contain the account information, adventure preferences, etc. |
| SCRUM-32 | Create Unit Tests For Features | Highest | Upon each sprint completion, unit tests need to be developed to ensure working features are being delivered and that the system is sound and meets desired functionality. |
| SCRUM-22 | Implement User Adventure Parameters Feature | Highest | The users adventure parameters/preferences are of high importance because that information is needed to feed back to the matching algorithm. |
| SCRUM-25 | Implement Matching Algorithm Feature | Highest | The matching algorithm is of high importance since that is the foundation of the functionality of the app. |
| SCRUM-16 | Implement Account Creation and Deletion Feature | High | Account creation/deletion is high since the user will need one to be able to use all the apps features. |
| SCRUM-13 | Implement Sign-On Feature | Medium | Sign-On is a medium priority since even in most apps and websites the user can sometimes use the service without signing in, features are limited. |
| SCRUM-18 | Implement Uploading Photos Feature | Medium | Photo upload is also medium priority since it’s a nice to have, the user can still be matched based off their adventure preferences alone. |
| SCRUM-27 | Implement in-app messaging feature | Medium | In-app messaging is medium priority since this is something that is commonly implemented across multiple different apps and services so there is existing knowledge/code to allow this to be implemented quicker. |
| SCRUM-14 | Implement single sign-on | Low | Single sign-on is nice to have so it’s low priority. |
| SCRUM-20 | Implement Social Media Profile Connection Feature | Low | Social media profile connection is a nice to have since the user can use the main features without it and since this feature will be presented to the user as optional, making it low priority. |

A special note on the medium priority work items, some of these are marked medium priority not due to their importance, but due to their implementation time. Features that are important but take less time fall in the middle range of the list.

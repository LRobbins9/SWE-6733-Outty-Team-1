# Definition of Ready  
Each backlog item will have a set criteria for the definition of ready. If a backlog item does not meet any of the criteria listed below, the item will remain  either in the backlog or be moved to the next sprint. 

<b> 1. Title </b>  
The backlog item contains a short and descriptive title that accurately reflects the task or feature.  

<b> 2. User Story Opening Sentence </b>  
The backlog item contains a user story sentence in the standard format of the following:  
- "As a [type of user], I want to [action user takes], so that [result of action]".   

<b> 3. Additional Details </b>  
The additional features in the backlog item will include any important information related to the task or feature such as constraints, implementation suggestions, edge cases, etc. Additional details serve as supplementary information to the developer assigned to the task or feature to aide them in development. 

<b> 4. Estimated Story Points </b>  
The estimated story point will determine the not only the effort of the task or feature, but also take into consideration the complexity and the time needed to complete the task or feature.

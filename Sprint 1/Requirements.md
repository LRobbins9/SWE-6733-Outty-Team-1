# Requirements

## Functional Requirements

1. The app shall allow users to create an account and log in.

2. The app shall use a suitable database for backend storage.

3. The app shall store user data in the selected database and encrypt sensitive user data.

4. The app shall allow users to create, edit, and delete their profiles.

5. The app shall allow users to upload photos.

6. The app shall collect user information, including adventure type, skill level, preferences, attitude, and mile range.

7. The app shall allow users to connect their profiles to Instagram and other social media platforms.

8. The app shall include a matching algorithm that recommends users based on adventure interests, skill level, behavior, and selected location range.

9. The app shall include a swipe-left and swipe-right interface for accepting or rejecting matches.

10. The app shall include internal messaging between matched users.

## Non-Functional Requirements

1. The app shall protect user data through encryption and authentication.

2. The app shall be available as either a mobile app or a web app.

3. The app shall be scalable.

4. The app shall be maintainable.

5. The app shall be reliable.

## Project Requirements

1. The project shall support Scrum-based project tracking using GitHub Projects, Jira, Trello, Azure DevOps, or another approved project management tool.

2. The project shall include project documentation in a public GitHub repository.

3. The project shall organize sprint materials in separate repository folders.

4. The project shall include an AI Usage Disclosure section in the README if AI tools are used.
